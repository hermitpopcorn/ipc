# 共有メモリ (Shared Memory) IPC デモ

## 共有メモリとは

**共有メモリ (Shared Memory)** は、複数のプロセスが同じメモリ領域にアクセスする仕組。
共通のメモリへの読み書きなので結構シンプルだが、レースコンディションなどによるデータ破損の危険性がある。

### フロー

1. **メモリの作成/オープン**
   - `shm_open()` で名前付き共有メモリ領域を作成
   - `ftruncate()` で初回はサイズを設定
     - `shm_open()` はサイズの引数がなく、0b の領域を作成する。そのため `ftruncate()` でサイズを設定する必要がある
   - `mmap()` でプロセスのアドレス空間にマッピング

2. **同期**
   - 複数プロセスからの同時アクセスを制御
   - **POSIX セマフォ** または **pthread mutex** を使用

3. **クリーンアップ**
   - `munmap()` でマッピング解除
   - `shm_unlink()` で共有メモリ削除

#### 問題の例

同期メカニズムが不十分だと、データ破損やデッドロックが発生してしまう。

例えば、複数プロセスが同時に起動して同時にセマフォやミューテックスを初期化しようとすると、未定義動作になってしまう。

```
プロセス A                  プロセス B
    |                          |
    |--- sem_open() ------>    |
    |                          |--- sem_open() ------>
    |--- pthread_mutex_init()  |
    |                          |--- pthread_mutex_init()
    |                          |
   [未定義動作]
```

両プロセスが同時に:
- `sem_open(O_CREAT)` を呼ぶ
- `pthread_mutex_init()` を呼ぶ

→ セマフォ/ミューテックスの二重初期化が発生
→ 未定義動作（Undefined Behavior）

#### 解決策

- マスタープロセスなどの唯一の初期化担当者という役割を１つのプロセスに指定し、先に起動させる

このデモでは:
- マスターのみがセマフォ・ミューテックスを初期化
- スレイブはマスター起動後に起動し、初期化済みのセマフォ/ミューテックスを利用
- マスター終了時にのみ `sem_unlink()` / `shm_unlink()` でクリーンアップ

## このデモについて

ブロードキャスト型の通信が可能な２つの実装例を作成した:

### 通信方式（単一カウンター・ブロードキャスト）

- **メッセージバッファ**: すべてのプロセスが同じ `message` に読み書き
- **単一カウンター**: すべてのプロセスが共有する `message_counter`
- **送信者情報**: `sender` フィールドにメッセージの送信元を記録

すべてのプロセス（マスター・複数のスレイブ）が:
- 自分のメッセージを共有メモリに書き込み
- カウンターをインクリメント
- 全員が同じカウンターをチェックして新しいメッセージを受信

→ 複数のスレイブが相互に通信でき、全員が全てのメッセージを見られる

### コード構造

- **common_shm.h**: 共通ロジックを集約したテンプレート関数
  - `run_common_process()`: 統一のイベントループ処理
  - 非ブロッキング stdin 読み込みで 100ms ごとにメッセージをチェック

- 実装（`shm_sem.cpp`, `shm_mutex.cpp`）
  - マスターのみ初期化とクリーンアップを担当
  - セマフォ・ミューテックスのロック掛け・解除処理関数を定義

この設計で複数のスレイブを起動可能:

```
ターミナル 1: ./shm_sem master
ターミナル 2: ./shm_sem slave1
ターミナル 3: ./shm_sem slave2
ターミナル 4: ./shm_sem slave3
```

### 1. POSIX セマフォ版 (`shm_sem.cpp`)

**POSIX セマフォ**を使った同期方式:

- セマフォは名前付きで、プロセス外で管理
- 複数のプロセスで共有できるグローバルロック

```
プロセス A              セマフォ              プロセス B
    |                     |                     |
  wait(sem)  ---------->  |  <-------- wait(sem)
    |                     |                     |
 [共有メモリアクセス]       |
    |                     |                     |
  post(sem)  ---------->  |                     |
    |                     |                     |
```

### 2. pthread mutex 版 (`shm_mutex.cpp`)

**pthread mutex**を共有メモリに埋め込む同期方式:

- ミューテックスを共有メモリ構造体の中に保持
- `PTHREAD_PROCESS_SHARED` 属性で複数プロセス対応

```
struct SharedData {
    pthread_mutex_t mutex;    <-- 共有メモリ内のミューテックス
    char data[...];
};
```

## 使用方法

```sh
# ビルド
make
```

```bash
# セマフォのデモ
# ターミナル1: マスター（初期化と管理）
./shm_sem master

# ターミナル2: スレイブ1
./shm_sem slave1

# ターミナル3: スレイブ2
./shm_sem slave2

# pthread mutex のデモ
# ターミナル1: マスター（初期化と管理）
./shm_mutex master

# ターミナル2: スレイブ1
./shm_mutex slave1

# ターミナル3: スレイブ2
./shm_mutex slave2
```

**操作方法**:
- 全プロセス: テキストを入力して Enter キーで送信
- 全プロセス: 新しいメッセージは 100ms ごとに自動的にチェックして表示
- 全プロセス: `quit` で終了
